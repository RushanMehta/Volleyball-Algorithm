Mathematical Formulation & Policy Derivation

This application treats a competitive volleyball set as a Finite-State Absorbing Markov Model optimized using Bellman Dynamic Programming backwards loops under standard Rally Scoring Rules (a point is sequentially awarded on every single rally outcome).

1. The Bellman Optimality Equation

Instead of merely evaluating fixed policies, the solver calculates the globally optimized set win equity at every single possible score state by selecting the action maximizing the immediate transition value:

$$V(u, t, \text{serve}) = \max_{a \in \{\text{TS}, \text{FL}\}} \sum_{u', t', s'} P(u', t', s' \mid u, t, \text{serve}, a) \cdot V(u', t', s')$$

2. Expected Rally Value (ERV) Metric

The performance indicator tracks net point distribution generated across an asset delivery cycle:

$$\text{ERV} = P(\text{Ace}) - P(\text{Error}) + P(\text{Poor})(1 - 2\cdot\text{Kill}_{\text{poor}}) + P(\text{Avg})(1 - 2\cdot\text{Kill}_{\text{avg}}) + P(\text{Perf})(1 - 2\cdot\text{Kill}_{\text{perf}})$$

Strategic Structural Boundary Simplifications
Semi-Markov Structural Compression Statement:
Real-world match sequences propagate across deep multi-contact trees:
$$\text{Serve Action} \longrightarrow \text{Pass Trajectory} \longrightarrow \text{Setter Location} \longrightarrow \text{Attack Selection} \longrightarrow \text{Point Outcome}$$ To ensure high structural reliability and eliminate overfitting to sparse high-school tracking data, this engine uses a robust semi-Markov approximation, compressing intermediate contacts into direct opponent defensive kill capabilities ($\text{Kill}_s$) mapped from pass execution categories.
